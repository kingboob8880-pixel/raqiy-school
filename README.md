# Онлайн-школа рукии

Курс подготовки заклинателя (раки) — от основ убеждённости (якын) и подлинных
хадисов о рукье до итоговой практики под супервизией и самостоятельного приёма
пациентов через систему диагностики и лечения **RUKYA Pro**.

Автор курса — лекарь Абу Мухаммад. Полная история решений, критерии отбора
контента и статус каждого модуля — в [`project.md`](./project.md), это рабочий
журнал проекта, а не просто план.

## Структура курса

11 модулей: от основ якына (Модуль 1) через диагностику без пальпаторных техник
(Модуль 4), направленное применение и изгнание духовных сущностей (Модули 5, 7),
до итоговой практики под супервизией наставника и работы в RUKYA Pro (Модули 8, 11).

Весь контент собран из 25 исходных материалов автора и прошёл триаж по трём
критериям (подробности — `project.md` §7/§9):
- прямая цитата аята/хадиса — включается как есть, с указанием источника;
- устоявшаяся шариатская практика — включается;
- авторская методика без хадисной основы (формулы-конструкторы, диагностика
  через надавливание на тело пациента, инструкции игнорировать страдание
  пациента) — **не сертифицируется**, помечается отдельно либо исключается.

Статус каждого модуля виден прямо на сайте (значок «Подтверждено шейхом» /
«Черновик, ждёт подтверждения» / «Вне сертифицируемой программы»).

## Структура репозитория

```
content/        # текст модулей и справочника — по одному .md на урок, с front matter (title/source/status)
design/         # design/tokens.json → tokens.css, base.css — единый источник стилей
pages/          # статические HTML-страницы + pages/js/ (данные модулей, рендер markdown, layout)
integration/    # клиентская интеграция с Firebase (auth, firestore, правила безопасности) + README по настройке
.claude/skills/ # скиллы Claude Code для дальнейшей разработки (в т.ч. /do — оркестратор фичей)
```

## Технологии

Статический сайт без сборщика — HTML/CSS/JS, готов к GitHub Pages как есть.
Markdown-контент рендерится в браузере (`marked.js` из CDN). Бэкенд —
**Firebase** (Authentication + Firestore), клиент подключается напрямую из
браузера, без собственного сервера.

## Запуск локально

```bash
python -m http.server 8000
# затем открыть http://localhost:8000/pages/index.html
```

## Перед публикацией / запуском для реальных учеников

1. Создать Firebase-проект и подставить конфиг — пошагово в
   [`integration/README.md`](./integration/README.md).
2. Подтвердить черновики контента (модули 3/4/5/7/8/11 и банк вопросов теста,
   `pages/js/quiz-data.js`) — они помечены как черновики намеренно и не
   выдаются за сертифицированные, пока автор курса их явно не одобрит.
3. Оплата — без платёжного шлюза: кнопка «Купить курс» ведёт в Telegram
   (`t.me/ruqoq`), доступ к продвинутому уровню открывается вручную
   переключателем в кабинете администратора после получения оплаты.

Полный статус реализации и то, что ещё не сделано — `project.md` §22.
