# UX Audit report template

Use this structure for the report. Write in the same language the user speaks.

```markdown
# UX Audit: [Site name / URL]
**Date:** [date]
**Auditor:** Claude (UX Audit Skill)

## Summary

**Overall score:** [X/10] — brief description
**Problems found:** [N] critical, [N] major, [N] minor

### Key findings
1. [Finding 1]
2. [Finding 2]
3. [Finding 3]

### Strengths
- [What's done well — with reasoning]

---

## Audit context

**Site:** [URL]
**Industry:** [industry]
**Target audience:** [description]
**Main conversion goal:** [goal]
**Pages audited:** [URL list]

---

## Block 1. Nielsen heuristics

### Problem 1.X: [Title]
- **Priority:** CRITICAL / MAJOR / MINOR
- **Heuristic:** [Which one is violated]
- **Where:** [Page / element]
- **What's wrong:** [Description]
- **Why it's a problem:** [Which principle is violated, how it affects the user, research reference if available]
- **What to do:** [Specific action]
- **Expected effect:** [What improves]

### Items without problems
- [1.X] [Title] — OK: [brief comment]

---

## Block 2. Conversion audit
[Same structure]

## Block 3. Content audit
[Same structure]

## Block 4. Technical audit
[Same structure]

## Block 5. Information architecture
[Same structure]

---

## Fix plan

### CRITICAL (immediately)
| # | Problem | Block | Action | Effort |
|---|---------|-------|--------|--------|

### MAJOR (next sprint)
| # | Problem | Block | Action | Effort |
|---|---------|-------|--------|--------|

### MINOR (backlog)
| # | Problem | Block | Action | Effort |
|---|---------|-------|--------|--------|

---

## Competitor benchmark (if conducted)

| Parameter | [Client] | [Competitor 1] | [Competitor 2] |
|-----------|----------|----------------|----------------|

---

## Methodology

Audit across 5 blocks: Nielsen heuristics, conversion, content, technical quality, information architecture. Each finding includes reasoning: violated principle, impact on the user, expected effect of the fix.
```
